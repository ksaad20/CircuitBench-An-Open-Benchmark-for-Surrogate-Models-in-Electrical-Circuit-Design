# Contributing to Circuit-Bench

Thank you for your interest in contributing to Circuit-Bench.

Circuit-Bench is an open-source benchmark for analog, digital, RF, power electronics, passive, and mixed-signal circuit analysis. Our goal is to build a reproducible, extensible, and community-driven benchmark that supports research, education, and industrial applications.

## Ways to Contribute

Contributions are welcome in many forms, including:

* New benchmark tasks
* Circuit datasets
* Dataset documentation
* Bug fixes
* Performance improvements
* New evaluation metrics
* Baseline implementations
* Tutorials and examples
* Documentation improvements
* Testing and validation
* Continuous integration enhancements

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes.
4. Run the full test suite.
5. Update documentation if necessary.
6. Submit a pull request.

## Development Workflow

Before opening a pull request:

* Ensure all automated tests pass.
* Run formatting and linting tools.
* Verify benchmark reproducibility.
* Update documentation for any new features.
* Include tests for new functionality whenever practical.

## Coding Standards

Please follow the project's coding conventions:

* Use descriptive variable and function names.
* Keep functions focused on a single responsibility.
* Write clear docstrings.
* Prefer readable code over clever code.
* Keep imports organized.
* Follow the repository's formatting and linting configuration.

## Documentation

When adding new functionality, update any relevant documentation, including:

* User guides
* Developer guides
* API documentation
* Examples
* Benchmark descriptions
* Dataset documentation

## Dataset Contributions

When contributing datasets, please provide:

* Dataset description
* Source and provenance
* License information
* Citation details
* Metadata
* Supported file formats
* Task mapping
* Preprocessing notes
* Validation status

Only contribute datasets that you have permission to share.

## Benchmark Contributions

New benchmark tasks should include:

* Problem description
* Input specification
* Output specification
* Evaluation metrics
* Reference implementation (if applicable)
* Example usage

## Testing

Before submitting your contribution, run the project's test suite and verify that your changes do not introduce regressions.

## Pull Requests

A good pull request should:

* Focus on a single logical change.
* Include a clear description.
* Reference related issues when appropriate.
* Include tests for new functionality.
* Update documentation where necessary.

Please respond to review comments constructively and be prepared to revise your contribution.

## Reporting Issues

When reporting a bug, include:

* A clear description of the problem.
* Steps to reproduce.
* Expected behavior.
* Actual behavior.
* Environment details (operating system, Python version, etc.).
* Relevant logs or error messages.

Feature requests should describe the motivation, proposed solution, and potential impact.

## Code of Conduct

Please interact respectfully with all contributors. Constructive discussions, thoughtful reviews, and collaborative problem-solving help maintain a healthy open-source community.

## Citation

If you use Circuit-Bench in academic work, please cite the associated publication(s). Citation information will be maintained in the repository and updated as new releases become available.

## License

By contributing to Circuit-Bench, you agree that your contributions will be distributed under the project's license unless otherwise stated.

Thank you for helping improve Circuit-Bench and supporting reproducible research in circuit benchmarking.
